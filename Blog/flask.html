How to Host a Simple HTML Webpage with Flask Server

This is a basic guide on how to create and host a simple HTML webpage. It includes instructions for local testing in a browser, as well as hosting options on platforms like Render, with the addition of a Flask server for local development.

1. Create Your HTML File

First, let's create the basic HTML file.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Simple Webpage</title>
    <link rel="stylesheet" href="styles.css">
    <script src="script.js" defer></script>
</head>
<body>
    <header>
        <h1>Welcome to My Simple HTML Webpage</h1>
    </header>
    <main>
        <p>This is a simple webpage.</p>
    </main>
</body>
</html>

2. Add a Stylesheet

Next, create a CSS file to style your webpage. Here's a simple styles.css file:

body {
    font-family: Arial, sans-serif;
    background-color: #f4f4f4;
    color: #333;
    margin: 0;
    padding: 20px;
}

header {
    text-align: center;
    background-color: #333;
    color: white;
    padding: 20px;
}

main {
    max-width: 800px;
    margin: auto;
    padding: 20px;
    background-color: white;
    border-radius: 8px;
}

3. Add JavaScript

Now, let's add some interactivity with JavaScript. Here's a simple script.js file:

document.addEventListener('DOMContentLoaded', function() {
    alert('Welcome to my webpage!');
});

4. Set Up a Flask Server

To serve these files dynamically using Flask, we will create a simple Flask server. Here's how you do it:

  • First, create a new directory for your Flask project (e.g., my-flask-server).
  • Inside this directory, initialize a new Python environment and install Flask:
    • python -m venv venv
source venv/bin/activate  # On Windows use 'venv\Scripts\activate'
pip install flask
  • Create a file called app.py and paste the following code into it:
    • from flask import Flask, send_from_directory
import os

app = Flask(__name__)

# Route for serving the home page
@app.route('/')
def home():
    return send_from_directory(os.getcwd(), 'index.html')

# Route for serving static files (CSS, JS, etc.)
@app.route('/<path:filename>')
def serve_static(filename):
    return send_from_directory(os.getcwd(), filename)

if __name__ == '__main__':
    app.run(debug=True)
  • Now, create a public folder in the same directory as app.py and move your index.html, styles.css, and script.js files into it.
  • Finally, start the Flask server by running the following command:
    • python app.py
  • Now, visit http://127.0.0.1:5000 in your browser to see your webpage served by the Flask server!

5. Why GitHub Pages Won't Work

Although GitHub Pages is a popular platform for hosting static websites, it **does not support dynamic server-side code** like Flask. This means you cannot use Flask to serve your dynamic content, handle routes, or interact with databases when hosting on GitHub Pages. GitHub Pages is designed to host only static files (HTML, CSS, JavaScript), and cannot execute backend server code like Flask.

For projects that require backend functionality, such as those built with Flask, you should use a platform like Render or Heroku, which support running full-stack applications, including Flask and other server-side code.

6. Host Your Webpage on Render

Render is a platform that supports Flask applications. Here's how to host your Flask server:

  • Sign up for a Render account at Render.
  • Create a new Web Service.
  • Choose "Web Service" and link your GitHub repository containing your Flask project.
  • Render will automatically deploy your site, and you'll be given a URL like https://my-webpage.onrender.com.

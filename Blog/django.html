How to Host a Webpage with Django

This guide explains how to use Django to create and host a simple webpage, including the advantages Django offers over Flask.

1. Set Up Your Django Project

Django is a high-level Python web framework that allows for rapid development and clean, pragmatic design. It includes many built-in features such as an admin panel, user authentication, and form handling, which makes it a powerful choice for larger projects. Here's how to get started:

  • First, create a new directory for your Django project (e.g., my-django-server).
  • Initialize a new Python environment and install Django:
    • python -m venv venv
source venv/bin/activate  # On Windows use 'venv\Scripts\activate'
pip install django
  • Now, create a new Django project by running:
    • django-admin startproject mysite

2. Create Your First Django App

In Django, an "app" is a web application that does something specific, such as a blog or a user authentication system. Let's create an app to serve our HTML page.

  • Inside your project directory, run the following command to create an app:
    • python manage.py startapp homepage
  • In the homepage directory, create a file called views.py to handle requests:
    • from django.shortcuts import render

def home(request):
    return render(request, 'index.html')
  • Now, set up a URL route to point to the view you just created in urls.py:
    • from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]

3. Serve Static Files

In Django, you can serve static files such as CSS, JavaScript, and images with ease. You'll need to set up a directory for your static files and configure Django to serve them.

  • In your settings.py file, configure the STATICFILES_DIRS setting:
    • STATICFILES_DIRS = [
    BASE_DIR / "static",
]
  • Make sure to place your index.html, styles.css, and script.js files inside the static directory.
  • In the urls.py file of your project, include the following line to serve static files during development:
    • from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    path('', views.home, name='home'),
] + static(settings.STATIC_URL, document_root=settings.STATIC_ROOT)

4. Start the Django Server

Now that you've set up the app and static file serving, it's time to run the server.

  • In your project directory, run the following command:
    • python manage.py runserver
  • Visit http://127.0.0.1:8000 in your browser to see your webpage served by Django!

5. Why Choose Django Over Flask?

While Flask is a lightweight framework that is great for small projects, Django is designed for larger projects and comes with many built-in features that can save time during development. Here's what Django offers:

  • Admin Interface: Django provides a powerful admin panel for managing your application's data.
  • Authentication: Django has built-in user authentication and authorization.
  • Form Handling: Django includes built-in tools for handling forms, including form validation and submission.
  • ORM (Object-Relational Mapping): Djangoâ€™s ORM allows you to interact with databases using Python code, making it easier to work with relational databases.
  • Scalability: Django is designed to scale efficiently for large applications with complex requirements.

6. Host Your Django Webpage on Render

Just like Flask, you can deploy your Django project on platforms like Render. Here's how to do it:

  • Sign up for a Render account at Render.
  • Create a new Web Service and link your GitHub repository containing your Django project.
  • Render will automatically deploy your site, and you'll be given a URL like https://my-django-app.onrender.com.
